# vz-summary-bot

Daily Verslo žinios summary, generated by GitHub Actions and emailed to your inbox.

## How it works

1. GitHub Actions cron fires at 04:30 UTC (≈07:30 Vilnius time year-round, give or take an hour for DST).
2. Script reads `state/last_run.json`, fetches the VŽ RSS feed, keeps articles published since that timestamp.
3. Filters by section: high-tier sections always pass; conditional sections pass a Gemini pre-filter for investing relevance; skip-tier sections are dropped.
4. Logs into vz.lt via Playwright, fetches up to 20 full article bodies, extracts main text with `trafilatura`.
5. Sends bodies to Gemini for structured extraction (JSON, one item per article).
6. Validates each item: its `source_quote_lt` must be a verbatim substring of the article body. Items that don't match are dropped.
7. Sends validated JSON to Gemini for HTML rendering.
8. Emails the HTML summary via Gmail SMTP.
9. Commits the updated `state/last_run.json` back to the repo so the next run picks up where this one stopped.

If anything fails, you get a separate failure email with the Python traceback.

## Setup (one-time, ~15 min)

### 1. Push this repo to GitHub

```bash
cd vz-summary-bot
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<your-username>/vz-summary-bot.git
git push -u origin main
```

Make the repo **private** — it doesn't contain secrets, but the email summaries you'll generate are personal.

### 2. Create a Gmail app password

You'll use this so the GitHub workflow can send mail from `timma717@gmail.com`.

1. Go to <https://myaccount.google.com/security>.
2. Enable 2-Step Verification if you haven't already.
3. Go to <https://myaccount.google.com/apppasswords>.
4. Create a new app password named "vz-summary-bot". Copy the 16-character password — you won't see it again.

### 3. Get a Gemini API key

1. Go to <https://aistudio.google.com/apikey>.
2. Create API key (free tier — `gemini-2.5-flash` is what we use).
3. Copy it.

### 4. Add GitHub Secrets

In your repo: **Settings → Secrets and variables → Actions → New repository secret**. Add each of these:

| Name             | Value                                              |
| ---------------- | -------------------------------------------------- |
| `VZ_EMAIL`       | Your VŽ login email (`timofej.gaivoronskij@stud.vilniustech.lt`) |
| `VZ_PASSWORD`    | Your VŽ password                                   |
| `GEMINI_API_KEY` | The key from step 3                                |
| `SMTP_USER`      | `timma717@gmail.com`                               |
| `SMTP_PASSWORD`  | The 16-char app password from step 2 (no spaces)   |
| `SMTP_TO`        | `timma717@gmail.com` (where summaries arrive)      |

### 5. First test run

1. Go to the **Actions** tab in your repo.
2. Click **Daily VŽ Summary** in the left sidebar.
3. Click **Run workflow** → **Run workflow**.
4. Wait ~3-5 minutes. Watch the logs in real time if you want.
5. Check `timma717@gmail.com` for the summary email.

If it fails, you'll get a failure email with the traceback. Most common first-run issue is a typo in one of the secrets.

After the first successful run, the cron will fire daily.

## Configuration

### Section tiers

Edit `main.py` constants `HIGH_TIER`, `CONDITIONAL_TIER`, `SKIP_TIER`. The keys are the URL-slug section names (the first path segment of a vz.lt article URL — e.g. `https://www.vz.lt/finansai/...` → `finansai`).

### Schedule

Edit `.github/workflows/daily.yml`'s `cron` value. Note: GitHub Actions cron uses UTC.

### Article cap

`MAX_ARTICLES` constant in `main.py`. Default 20.

### Reset / replay

To re-process a specific time window, manually edit `state/last_run.json` and push. The next workflow run will fetch everything published since that timestamp (capped at 20).

## Troubleshooting

**Login fails** ("no 'Atsijungti' marker")
VŽ may have changed their login form selectors. Check screenshots from the probe script (`vz_probe_browser.py` in your Desktop), update `LOGIN_URL` / selectors in `login_and_fetch()` of `main.py`.

**Articles all skipped as paywalled**
The `PAYWALL_TEXT_MARKERS` may be hitting in unrelated places of the extracted body. Open one of the failing article URLs while logged in, check what `trafilatura.extract` returns for it (run locally), and adjust the markers.

**Gemini returns invalid JSON**
The `strip_fences` and JSON-salvage logic in `gemini_extract` handles common cases. If it persistently fails, lower `temperature` further or split into smaller batches.

**Workflow can't push state changes**
Ensure repo settings → Actions → General → Workflow permissions is set to **Read and write**.

## Files

- `main.py` — pipeline (RSS → login → scrape → Gemini → validate → email)
- `requirements.txt` — Python deps
- `.github/workflows/daily.yml` — cron + dispatch
- `state/last_run.json` — last successful run timestamp (committed by workflow)
